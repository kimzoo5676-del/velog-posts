# 리눅스 공부를 시작하며 
---
## 1. 가상머신 소프트웨어란?

가상머신 소프트웨어란, 지금 내가 쓰는 **macOS(또는 Windows)를 그대로 유지하면서** 그 위에 여러 대의 리눅스 서버를 동시에 운영하는 효과를 낼 수 있는 소프트웨어다.

아래 그림처럼 1대의 PC에서 리눅스 서버A, 리눅스 서버B, 리눅스 클라이언트, Windows 클라이언트까지 — 총 4개의 게스트 OS를 동시에 띄울 수 있다. 

원래 내 PC의 macOS까지 합치면 **1대에서 5개의 운영체제가 동시에 가동**되는 셈이다.

---

## 2. 핵심 용어: 호스트 OS vs 게스트 OS

가상머신을 공부하면서 가장 먼저 구분해야 하는 개념이다.

| 구분 | 설명 |
|------|------|
| **호스트 OS (Host OS)** | 물리 하드웨어 위에 직접 설치된 주인 운영체제 |
| **게스트 OS (Guest OS)** | 가상머신 위에서 돌아가는 손님 운영체제 |

호스트 OS는 실제 하드웨어(CPU, RAM, 네트워크 카드)를 직접 사용하고, 게스트 OS는 그 자원을 **소프트웨어적으로 쪼개서 빌려 쓰는** 구조다.

예를 들어, 내 맥북 RAM이 16GB라면 게스트 A에게 2GB, 게스트 B에게 2GB를 할당하고, 남은 12GB로 내 macOS를 돌리는 식이다.

---

## 3. 멀티부팅 vs 가상머신 — 뭐가 다를까?

처음 접할 때 가장 헷갈리는 개념이 **멀티부팅**과 **가상머신**의 차이다.

### 멀티부팅 (Multi-Booting)
- 하드디스크의 파티션을 분할해 여러 OS를 설치
- **한 번에 하나의 OS만 가동** 가능
- OS를 바꾸려면 컴퓨터를 재부팅해야 함

### 가상머신 (Virtual Machine)
- 파티션 분할 없이 소프트웨어로 독립된 컴퓨터 환경을 생성
- **동시에 여러 OS를 가동** 가능
- OS 간에 **가상 네트워크로 통신**도 가능

| 구분 | 멀티부팅 | 가상머신 |
|------|----------|----------|
| 동시 실행 | ❌ 한 번에 하나 | ✅ 동시에 여러 개 |
| OS 간 네트워크 | ❌ 불가능 | ✅ 가능 |
| 실습 환경 구성 | ❌ 서버-클라이언트 통신 테스트 불가 | ✅ 실무 환경 재현 가능 |
| 안전성 | ❌ 망가지면 복구 어려움 | ✅ 스냅샷으로 즉시 복구 |

## 4. 가상 네트워크 구조 — 내 노트북 안의 전산실

아래 그림(책의 그림 1-3)이 이번 실습 전체의 핵심 구조다.

```
인터넷
  │
네트워크 카드 (호스트 컴퓨터)
  │  ← 기존 IP (호스트 OS가 사용)
  │
가상 라우터 (Gateway)  ← 가상머신 소프트웨어가 생성
  │
  ├── 리눅스 서버A   (가상 IP)
  ├── 리눅스 서버B   (가상 IP)
  ├── 리눅스 클라이언트 (가상 IP)
  └── Windows 클라이언트 (가상 IP)
```

### 핵심 포인트

**기존 IP vs 가상 IP**
- 호스트 OS는 실제 공유기(또는 랜선)에서 받아온 **기존 IP**를 사용한다.
- 게스트 OS들은 가상머신 소프트웨어가 내부적으로 부여한 **가상 IP**를 사용한다.
- 외부 인터넷에서 보면 그냥 컴퓨터 1대지만, 내부에서는 5개.

**가상 라우터(Gateway)의 역할**
- 집의 공유기가 하는 역할을 가상 환경에서 소프트웨어가 대신한다.
- 4대의 게스트 OS는 이 가상 라우터를 통해 서로를 "옆 컴퓨터"로 인식하고 데이터를 주고받는다.
- 나중에 리눅스 설정 파일에서 `GATEWAY=192.168.x.x` 같은 줄을 보게 될 텐데, 그때 이 구조를 떠올리면 된다.

## 5. DevOps 관점에서 이 구조가 중요한 이유

지금 설정하는 이 구조는 단순한 실습용이 아니다. **현업 인프라의 축소판**이다.

### 서버를 나누는 이유 — 가용성과 보안
실무에서는 웹 서버, DB 서버, 모니터링 서버를 **역할별로 독립된 컴퓨터에 분리**한다.

- 가용성: "한 서버가 죽어도 나머지는 살아야 한다"
- 보안: "외부에 노출되는 서버와 내부 전용 서버를 분리한다"

그림 1-3처럼 서버A, 서버B, 클라이언트를 나누는 것도 같은 원리다.

### 하이퍼바이저(Hypervisor)
VMware 같은 가상머신 소프트웨어를 전문 용어로 **하이퍼바이저**라고 부른다.  
AWS, Azure 같은 클라우드 서비스도 결국 거대한 물리 서버 위에 이 하이퍼바이저 기술을 올려서 우리에게 **가상 서버(EC2 등)를 빌려주는** 원리다.

### Docker와의 연결
나중에 배우게 될 **Docker(컨테이너)**는 VM에서 한 발 더 나아간 기술이다.

- VM: 운영체제 전체를 가상화 → 무겁지만 완전한 격리
- Docker: OS를 공유하면서 프로세스 단위만 격리 → 가볍고 빠름

VM의 원리를 잘 잡아두면 Docker를 배울 때 "아, 이게 더 가볍게 만든 거구나" 하고 바로 연결된다.

## 6. 실습 도구: VMware

실습에서는 **VMware Workstation Pro / VMware Fusion Pro**를 사용한다.

- **VMware Workstation Pro**: Windows/Linux 사용자용
- **VMware Fusion Pro**: macOS 사용자용

### 설치 전에 미리 알아두면 좋은 용어

| 용어 | 설명 |
|------|------|
| **ISO 파일** | 리눅스를 설치하기 위한 '가상 설치 CD'. 이 파일을 가상머신에 넣어야 설치가 시작됨 |
| **VMDK** | 가상머신의 '하드디스크' 파일. 게스트 OS 안의 모든 데이터가 이 파일 하나에 저장됨 |
| **NAT** | 호스트 PC의 IP를 빌려 게스트가 인터넷에 나가는 방식. 설정이 간편해 기본값으로 주로 사용 |
| **Bridge** | 게스트가 공유기에서 직접 독립 IP를 받아오는 방식. 호스트와 동등한 네트워크 위치 |
| **스냅샷(Snapshot)** | 현재 VM 상태를 저장해두는 기능. 설정을 잘못 건드려 망가져도 클릭 한 번으로 복구 가능 |
