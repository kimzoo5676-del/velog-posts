# 물리 계층과 데이터 링크 계층

## 왜 이 두 계층을 같이 볼까?

OSI 모델을 보면 물리 계층(1계층)과 데이터 링크 계층(2계층)은 서로 다른 계층이다. 그런데 이 둘을 따로따로 설명하지 않고 묶어서 설명하는 경우가 많다. 왜일까?

바로 **이더넷(Ethernet)** 이라는 기술이 두 계층 모두에 걸쳐 동작하기 때문이다.

오늘날 유선 LAN 환경이라면 거의 대부분 이더넷 기반이다. 물리 계층에서는 이더넷 규격의 케이블을, 데이터 링크 계층에서는 이더넷 프레임 형식을 사용한다. 이 두 계층이 같은 기술을 공유하고 있으니 함께 이해하는 게 자연스럽다.

---

## 이더넷(Ethernet)

> **이더넷이 뭐야?**

현대 유선 LAN 환경에서 가장 대중적으로 사용되는 네트워크 기술이다. 단순히 "케이블로 연결하는 것"이 아니라, 다음과 같은 내용들이 모두 정의된 표준이다.

- 어떤 통신 매체(케이블)를 사용할지
- 프레임의 형태가 어떻게 생겼는지
- 프레임을 어떻게 주고받는지

이더넷의 표준은 **IEEE 802.3** 이라는 이름으로 관리된다. 802.3u, 802.3ab처럼 세부 표준들이 계속해서 추가되고 있으며, 표준에 따라 지원되는 장비, 케이블 종류, 전송 속도 등이 달라질 수 있다.

---

## 물리 계층 — 신호를 주고받는 계층

### 통신 매체 표기 읽는 법

케이블 스펙을 보면 `100BASE-T`, `10GBASE-SR` 같은 이름을 자주 마주친다. 처음엔 암호처럼 보이지만 규칙만 알면 쉽게 읽힌다.

```
전송속도  BASE  추가특성
  100    BASE    T
  10G    BASE    SR
```

| 부분 | 의미 |
|------|------|
| **전송속도** | 숫자만 있으면 Mbps, 뒤에 G가 붙으면 Gbps |
| **BASE** | 베이스밴드(BASEband)의 약자. 비트 신호를 변환해 전송하는 방식 |
| **추가특성** | 통신 매체 종류, 최대 거리, 인코딩 방식 등 |

추가특성에서 통신 매체 종류는 알파벳으로 구분한다.

| 표기 | 케이블 종류 |
|------|------------|
| **C** | 동축 케이블 |
| **T** | 트위스티드 페어 케이블 |
| **S** | 단파장 빛을 사용하는 광섬유 케이블 |
| **L** | 장파장 빛을 사용하는 광섬유 케이블 |

즉, `100BASE-T`는 "100Mbps 속도의, 트위스티드 페어 케이블을 쓰는 이더넷"이고, `1000BASE-SX`는 "1Gbps 속도의, 단파장 광섬유를 쓰는 이더넷"이다.

### NIC — 신호와 데이터를 이어주는 장치

> **케이블로 연결만 하면 통신이 되는 건가?**

케이블은 전기 신호나 빛 신호를 운반한다. 하지만 컴퓨터는 그 신호를 그대로 이해하지 못한다. 사람에게 전달하려면 번역이 필요하듯, 이 신호를 컴퓨터가 이해할 수 있는 정보로 변환해주는 장치가 필요하다.

그 역할을 하는 것이 바로 **NIC(Network Interface Controller)** 다.

NIC는 카드 형태로 꽂는 것도 있고, USB로 연결하는 것도 있고, 마더보드에 아예 내장된 것도 있다. 우리가 흔히 "랜카드"라고 부르는 게 NIC다.

NIC에는 중요한 특징이 하나 더 있다. 바로 **MAC 주소**가 여기에 부여된다는 점이다. MAC 주소는 네트워크 인터페이스마다 할당되는 물리적 주소인데, NIC가 바로 그 네트워크 인터페이스 역할을 수행한다.

---

### 트위스티드 페어 케이블

> **가장 흔하게 보이는 랜선**

구리 선으로 전기 신호를 주고받는 케이블이다. 우리가 흔히 "랜선"이라고 부르는 것의 정식 명칭이다.

케이블 본체를 보면 구리 선 두 가닥이 서로 꼬여 있다. 꼬아놓은 이유가 있다. 구리 선에 전류가 흐르면 전자기 간섭(노이즈)이 생기는데, 서로 반대 방향으로 꼬아 감으면 간섭이 상쇄된다.

> **근데 꼬는 것만으로 노이즈가 다 해결돼?**

완전히는 아니다. 그래서 **차폐(shielding)** 라는 방법도 함께 사용한다. 구리 선을 그물 모양 철사(**브레이드 실드**)나 은박지 같은 포일(**포일 실드**)로 감싸 외부 노이즈를 막는 것이다.

```
UTP  ─  차폐 없음 (Unshielded)
FTP  ─  포일 실드로 감쌈 (Foiled)
STP  ─  브레이드 실드로 감쌈 (Shielded)
```

케이블 이름은 `XX/YTP` 형태로 표기한다. `XX`는 외부 전체를 감싸는 실드 종류, `Y`는 각 구리 선 쌍을 감싸는 실드 종류다.

예: `S/FTP` 케이블, `SF/FTP` 케이블

또한 케이블 성능은 **카테고리(Cat)** 로 구분한다. `Cat5`, `Cat6`, `Cat7`처럼 숫자가 높을수록 더 높은 대역폭과 전송 속도를 지원한다.

---

### 광섬유 케이블

> **빛으로 데이터를 보낸다**

트위스티드 페어 케이블이 전기 신호를 쓴다면, **광섬유 케이블**은 빛(광신호)을 이용한다.

빛은 전기보다 훨씬 빠르고, 노이즈 간섭도 거의 없으며, 신호 손실 없이 먼 거리를 이동할 수 있다. 그래서 대륙 간 네트워크 연결처럼 장거리 통신에 광섬유 케이블이 사용된다.

케이블 내부를 들여다보면 머리카락처럼 가느다란 **광섬유**가 들어 있다.

```
┌─────────────────────────┐
│  클래딩(Cladding)      │  ← 빛을 가두는 역할
│   ┌───────────────┐     │
│   │  코어(Core)  │     │  ← 빛이 실제로 흐르는 부분
│   └───────────────┘     │
└─────────────────────────┘
```

코어의 지름에 따라 두 종류로 나뉜다.

| 종류 | 특징 |
|------|------|
| **싱글 모드** (노랑/파랑) | 코어가 가늘어 빛 경로가 하나. 신호 손실이 적어 **장거리**에 적합. 장파장 빛 사용. 비쌈. |
| **멀티 모드** (오렌지/아쿠아) | 코어가 두꺼워 빛 경로가 여러 개. 신호 손실이 커 **근거리**에 적합. 단파장 빛 사용. |

## 데이터 링크 계층 — 같은 네트워크 안에서의 통신

물리 계층이 "신호를 어떻게 전달하느냐"의 문제를 다뤘다면, 데이터 링크 계층은 "같은 네트워크 안에서 누구에게 어떻게 전달하느냐"를 다룬다.

### 이더넷 프레임

> **데이터 링크 계층에서 주고받는 단위가 뭐야?**

이 계층의 PDU(Protocol Data Unit)는 **프레임(Frame)** 이다. 유선 LAN 환경에서는 이더넷 표준에 따른 **이더넷 프레임** 형식을 사용한다.

프레임의 구조를 살펴보자.

```
┌──────────┬────────────┬────────────┬──────────┬──────────────┬─────┐
│ 프리앰블  │ 수신지      │ 송신지      │ 타입/    │   데이터      │ FCS │
│  8바이트  │ MAC 주소   │ MAC 주소   │  길이    │ (46~1500B)   │     │
│          │  6바이트   │  6바이트   │  2바이트  │              │     │
└──────────┴────────────┴────────────┴──────────┴──────────────┴─────┘
   ◀──────────────── 헤더 ────────────────▶◀── 페이로드 ──▶◀트레일러▶
```

각 필드가 무슨 역할을 하는지 하나씩 살펴보자.

**① 프리앰블 (8바이트)**

이더넷 프레임의 시작을 알리는 신호다. 앞 7바이트는 `10101010`, 마지막 1바이트는 `10101011`로 구성된다. 수신 측은 이 패턴을 보고 "아, 이제 프레임이 온다"는 걸 인식하고 송수신 타이밍을 맞춘다.

**② MAC 주소 (각 6바이트)**

수신지와 송신지를 특정하는 물리적 주소다. 네트워크 인터페이스(NIC)마다 고유하게 부여되며, 일반적으로 변경되지 않는다.

**③ 타입/길이 (2바이트)**

필드 값이 `1500(0x05DC)` 이하면 데이터 길이를, `1536(0x0600)` 이상이면 상위 계층 프로토콜 타입(이더타입)을 나타낸다.

```
0x0800  →  IPv4
0x86DD  →  IPv6
0x0806  →  ARP
```

**④ 데이터 (46~1500바이트)**

실제로 전달할 내용이다. 네트워크 계층의 패킷(헤더 + 데이터)이 여기 담긴다. 최소 크기인 46바이트에 못 미치면 패딩을 채워 맞춘다.

**⑤ FCS (Frame Check Sequence)**

수신한 프레임에 오류가 있는지 확인하는 트레일러다. **CRC(순환 중복 검사)** 값이 담겨 있다.

송신 측에서 프레임을 보낼 때 CRC 값을 계산해 FCS에 넣고, 수신 측에서도 똑같이 계산한 뒤 비교한다. 값이 일치하면 오류 없음, 불일치하면 해당 프레임을 폐기한다.

## 허브 vs 스위치 — 어떻게 진화했을까?

### 허브 — 무조건 다 뿌린다

**허브(Hub)** 는 여러 호스트를 연결하는 장치다. 물리 계층 장비라서 MAC 주소 같은 개념이 없다. 그래서 특정 호스트에게만 프레임을 보낼 수 없고, 받은 신호를 연결된 **모든 포트로 그대로 내보낸다.**

> **그럼 받은 호스트들은 다 읽어?**

아니다. 프레임을 받은 각 호스트들은 데이터 링크 계층에서 MAC 주소를 확인하고, 자신과 관련 없는 프레임은 폐기한다. 낭비처럼 보이지만, 허브는 이게 전부다.

> **또 다른 문제는 없어?**

있다. 허브는 **반이중(Half Duplex) 모드**로 통신한다. 무전기처럼 송신과 수신을 동시에 할 수 없어 번갈아가며 해야 한다.

그래서 두 호스트가 동시에 신호를 보내면 **충돌(Collision)** 이 발생한다. 허브에 연결된 모든 호스트는 같은 **콜리전 도메인(Collision Domain)** 에 속하기 때문에, 호스트 수가 많아질수록 충돌 가능성도 높아진다.

> **충돌 문제를 어떻게 해결해?**

**CSMA/CD 프로토콜**이 그 역할을 한다. 이름이 길어 보이지만 세 단계로 나눠 생각하면 쉽다.

```
① Carrier Sense (캐리어 감지)
   → 지금 통신 매체를 누가 쓰고 있는지 먼저 확인한다

② Multiple Access (다중 접근)
   → 여러 호스트가 동시에 접근하는 상황을 허용하되

③ Collision Detection (충돌 검출)
   → 충돌이 감지되면 즉시 전송을 중단하고,
      잼(Jam) 신호를 보낸 뒤 랜덤한 시간을 기다렸다가 재전송한다
```

### 스위치 — 필요한 곳에만 보낸다

> **애초에 필요한 포트로만 보내면 되잖아?**

맞다. 그게 바로 **스위치(Switch)** 가 하는 일이다. 데이터 링크 계층 장비라서 L2 스위치라고도 부른다.

스위치가 허브보다 나은 이유는 두 가지다.

1. **MAC 주소 학습**: 어떤 포트에 어떤 호스트가 연결되어 있는지 기억한다.
2. **전이중(Full Duplex) 통신**: 송신과 수신을 동시에 할 수 있다. 충돌이 발생하지 않는다.

> **MAC 주소를 어떻게 학습해?**

스위치는 **MAC 주소 테이블**이라는 표를 메모리에 유지한다. 처음엔 이 표가 비어 있다.

아래 예시로 학습 과정을 따라가 보자.

```
포트 1 ─ [호스트 A]
포트 2 ─ [호스트 B]
포트 3 ─ [호스트 C]
포트 4 ─ [호스트 D]
```

**Step 1**: 호스트 A가 호스트 C에게 프레임을 전송한다.

스위치는 프레임의 **송신지 MAC 주소**를 읽고 "포트 1 = A의 MAC 주소"를 테이블에 기록한다.

**Step 2**: 아직 C의 주소를 모르니, 포트 1을 제외한 모든 포트로 프레임을 전송한다. 이를 **플러딩(Flooding)** 이라고 한다.

**Step 3**: B와 D는 자신과 무관한 프레임이므로 폐기한다. C는 응답 프레임을 보내고, 스위치는 이를 통해 "포트 3 = C의 MAC 주소"도 테이블에 기록한다.

**Step 4**: 이후부터는 A → C 통신 시 스위치가 포트 3으로만 프레임을 **포워딩(Forwarding)** 하고, B와 D로 나가는 포트는 **필터링(Filtering)** 한다.

```
MAC 주소 테이블 (학습 후)
┌────────┬─────────────────┐
│  포트   │   MAC 주소       │
├────────┼─────────────────┤
│  포트 1 │  AA:AA:AA:AA:AA │  ← 호스트 A
│  포트 3 │  CC:CC:CC:CC:CC │  ← 호스트 C
└────────┴─────────────────┘
```

참고로 테이블에 등록된 포트에서 일정 시간 동안 프레임이 오지 않으면 해당 항목은 자동으로 삭제된다. 이를 **에이징(Aging)** 이라고 한다.

### VLAN — 하나의 스위치로 여러 네트워크처럼

> **같은 스위치에 연결된 호스트들을 서로 격리시키고 싶다면?**

예를 들어 같은 사무실에 개발부와 총무부가 있는데, 두 부서의 네트워크를 분리하고 싶다. 물리적으로 스위치를 두 대 사는 게 가장 직관적이지만, 비용과 관리가 부담된다.

**VLAN(Virtual LAN)** 은 **하나의 물리적 스위치로 마치 여러 대의 스위치**가 있는 것처럼 논리적으로 LAN을 나누는 기술이다.

VLAN으로 나뉜 호스트들은 같은 스위치에 연결되어 있어도 서로 직접 통신할 수 없다. 상위 계층 장비(라우터 등)를 거쳐야만 통신이 가능하다.

> **VLAN을 구성하는 방법은?**

**① 포트 기반 VLAN**

**정적(Static) VLAN**이라고도 부름

스위치의 포트에 VLAN을 미리 할당해두는 방식이다. "포트 1~4는 개발부, 포트 5~8은 총무부" 식으로 설정한다.

스위치 여러 대에 걸쳐 같은 VLAN을 운영하려면 **VLAN 트렁킹**을 사용한다. 스위치 간 연결을 위한 특별한 **트렁크 포트**를 통해 여러 VLAN의 프레임이 오가는데, 이때 프레임이 어느 VLAN 소속인지 식별하기 위해 **802.1Q 태그**가 프레임 헤더에 추가된다.


**② MAC 기반 VLAN**

**동적(Dynamic) VLAN**이라고도 부름

포트가 아니라 호스트의 MAC 주소에 따라 VLAN을 결정하는 방식이다. 호스트가 다른 포트에 연결되더라도 동일한 VLAN에 속할 수 있다.

---

## 정리

| | 허브 | 스위치 |
|--|------|--------|
| **계층** | 물리 계층 (1계층) | 데이터 링크 계층 (2계층) |
| **주소 인식** | ❌ 없음 | ✅ MAC 주소 학습 |
| **전송 방식** | 모든 포트에 플러딩 | 특정 포트에만 포워딩 |
| **통신 모드** | 반이중 (충돌 발생) | 전이중 (충돌 없음) |
| **충돌 해결** | CSMA/CD | 필요 없음 |

물리 계층은 "신호를 어떻게 실어 나를까", 데이터 링크 계층은 "같은 네트워크 안에서 누구에게 어떻게 전달할까"에 집중한다. 다음 글에서는 네트워크 계층으로 올라가 서로 다른 네트워크 간 통신, 즉 IP와 라우팅을 살펴볼 것이다.
